# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in Eva Board, please report it through
[GitHub Security Advisories](https://github.com/EvaEverywhere/eva-board/security/advisories/new)
on this repository.

**Do not** open a public issue for security bugs.

## What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response timeline

We will acknowledge your report within **48 hours** and aim to provide a fix or
mitigation plan as quickly as possible.

## Scope

This policy covers the Eva Board codebase (backend, frontend, infrastructure
configs). Third-party dependencies should be reported to their respective
maintainers.
